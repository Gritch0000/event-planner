# 📅 Планувальник подій (Full-Stack)

**Тема:** Організація подій з учасниками, датами та нагадуваннями.  
Навчальний проєкт, розроблений під час практики для демонстрації навичок створення клієнт-серверних веб-застосунків.

---

## ✨ Реалізований функціонал

- **Управління подіями (CRUD):** Створення нових подій, перегляд списку, редагування місця проведення та видалення.
- **Учасники:** Можливість "приєднатися" до події (збір імені та email). Дані учасників зберігаються в `localStorage` браузера.
- **Розумні нагадування:** Автоматичний розрахунок дат. Якщо до події залишилося 3 дні або менше, на картці з'являється яскрава мітка **«🔔 Скоро!»**.
- **Сучасний UI:** Адаптивний дизайн на базі CSS Flexbox/Grid, зручні модальні вікна та стани завантаження.

---

## 🛠 Технології

- **Backend:** Nest.js, TypeScript
- **Frontend:** React, Vite, React Router
- **Зв'язок та дані:** Axios, LocalStorage
- **Стилі:** CSS (Flexbox/Grid)

---

## 🚀 Як запустити проєкт локально

Для роботи застосунку потрібно запустити обидва сервери (API та Клієнт) у двох окремих терміналах.

### 1. Запуск Backend (API)
Відкрийте перший термінал, перейдіть у папку `backend` та запустіть сервер:
```bash
cd backend
npm install
npm run start:dev


Сервер запуститься на порту `:3000`

### 2. Запуск Frontend (Клієнт)
Відкрийте другий термінал, перейдіть у папку `frontend` та запустіть інтерфейс:

```bash
cd frontend
npm install
npm run dev

Відкрийте у браузері посилання, яке видасть термінал (зазвичай http://localhost:5173).

## 🏗 Структура даних

### **Event (Подія)**
| Поле | Тип | Опис |
| :--- | :--- | :--- |
| **id** | `number` | Унікальний ідентифікатор події |
| **title** | `string` | Назва події |
| **description** | `string` | Опис заходу |
| **date** | `string` | Дата проведення |
| **location** | `string` | Місце проведення |

### **Participant (Учасник)**
| Поле | Тип | Опис |
| :--- | :--- | :--- |
| **id** | `number` | Унікальний ідентифікатор учасника |
| **name** | `string` | Повне ім’я учасника |
| **email** | `string` | Електронна пошта |
| **eventId** | `number` | ID події, до якої він прив'язаний |

## 🔌 API Ендпоінти

| Метод | Шлях | Опис |
| :--- | :--- | :--- |
| `GET` | `/events` | Отримати список усіх подій |
| `POST` | `/events` | Створити нову подію |
| `PATCH` | `/events/:id` | Змінити дані події (локацію) |
| `DELETE` | `/events/:id` | Видалити подію |
